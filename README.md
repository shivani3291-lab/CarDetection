End-to-end deep learning system for **Stanford Cars** (196 classes, ~16K images) with MLOps and a Streamlit dashboard.

## Fresh machine setup

### 1. Clone and create environment

```bash
conda env create -f environment.yml
conda activate car-detect
# or: pip install -r requirements.txt
```

### 2. Add the dataset

Copy `Car Images.zip` and `Car names and make.csv` into the project root, then:

```bash
python scripts/setup_data.py --zip "Car Images.zip"
python -m src.data.preprocess
python -m src.data.validate
python scripts/run_eda.py
```

### 3. Train models

```bash
# Quick baseline (~15 epochs)
python -m src.train.classifier_train --config configs/classifier_custom_cnn.yaml

# Transfer learning (recommended)
python -m src.train.classifier_train --config configs/classifier_resnet50.yaml
python -m src.train.classifier_train --config configs/classifier_efficientnet.yaml

# Detection
python -m src.train.detection_train --config configs/detection_fasterrcnn.yaml

# YOLOv8
python scripts/generate_yolo_dataset.py
python scripts/train_yolo.py

mlflow ui
```

### 4. Export and run the app

```bash
python -m src.mlops.export_onnx
streamlit run app.py
```

### 5. Full pipeline (one command)

```powershell
# Windows
powershell -File scripts/run_pipeline.ps1
```

```bash
# Linux/macOS
bash scripts/run_pipeline.sh
```

## Project structure

```
CarDetection/
├── app.py                  # Streamlit entrypoint
├── pages/                  # Multi-page dashboard
├── scripts/                # setup_data, run_eda, train_yolo, run_pipeline
├── src/
│   ├── data/               # Loading, augmentation, validation
│   ├── models/             # CNN, ResNet, Faster R-CNN, YOLO
│   ├── train/              # Training scripts
│   ├── evaluate/           # Metrics, evaluation reports
│   ├── mlops/              # ONNX export, promotion, drift
│   └── serve/              # Inference + Streamlit session
├── configs/                # Experiment YAML configs
├── data/
│   ├── class_names.json    # committed
│   ├── raw/                # extract Car Images.zip here (not in git)
│   └── processed/          # generated by preprocess
├── models/                 # trained weights + ONNX (not in git)
├── notebooks/01_eda.ipynb
├── dvc.yaml
└── params.yaml
```

## Streamlit Cloud deployment

1. Push repo to GitHub
2. Deploy at [share.streamlit.io](https://share.streamlit.io) with `app.py`
3. If ONNX is too large for git, host it and set `MODEL_URL` in Secrets


## Phases

| Phase | Deliverable |
|---|---|
| 0 | Repo structure, CI, Docker, `environment.yml` |
| 1 | Preprocess, GE validation, EDA scripts |
| 2 | CNN + ResNet + EfficientNet + MLflow |
| 3 | Faster R-CNN + YOLOv8 + mAP evaluation |
| 4 | CI/CD, ONNX export, model promotion, Evidently drift |
| 5+7 | Streamlit multi-page app with ONNX/detection inference |
| 6 | README, MODEL_CARD, notebooks |

## Notes

- Bounding boxes default to full-image when `.mat` annotations are not provided.
- Validation allows 99% train class coverage (195/196 classes is expected for this zip).
- GPU strongly recommended for ResNet, Faster R-CNN, and YOLO training.
